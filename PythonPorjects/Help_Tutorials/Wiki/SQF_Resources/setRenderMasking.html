    setRenderMasking


    Introduced in

    Version:

    VBS2 2.12

    VBS4 Description (VBS4 22.2+)

    Description:

    Enables / disables classification rendering. The command can apply the default VBS4 or custom classification rendering, using an XML file.

    The format of the classification-rendering XML file is:

    <SensorClassification>
    <!-- Mandatory root "unlabeled" class -->
    <PropertyClass name="unlabeled" color="255,0,255">
    <!-- Various P3D named properties -->
    <PropertyClass name="building" color="200,200,0"/>
    <PropertyClass name="object" color="0,100,100" />
    <PropertyClass name="streetlamp" color="255,0,0"/>
     
    <!-- VBS Blue special classes -->
    <PropertyClass name="bush" color="0,200,60"/>
    <PropertyClass name="tree" color="10,100,10"/>
    <PropertyClass name="grass" color="0,80,0"/>
    <PropertyClass name="ground" color="100,100,100" />
    <!-- Examples of material classes -->
    <MaterialClass name="sky" color="133,133,255"/>
    <MaterialClass name="water" color="0,0,200"/>
    <MaterialClass name="road" color="175,175,175" />
    <MaterialClass name="asphalt" color="200,200,200"/>
    </PropertyClass>
    </SensorClassification>

    Everything is defined under <SensorClassification>, using the <PropertyClass> and / or <MaterialClass> tags. The render-mask color attribute is defined in the RGB format. For valid / recognized VBS4 name attribute values, contact support@bisimulations.com .

    VBS4 Syntax (VBS4 22.2+)

    Syntax:

    setRenderMasking [enable, classification_rendering_file]

    Parameters:

    • enable: Boolean - If set to true, classification rendering is enabled.

    • classification_rendering_file: String - Custom classification-rendering XML file (for the format specification, see the VBS4 Description) (optional, default: VBS4 Installation\systems\VBSBlue\vbs\blue\sensorplugins\SensorClassificationParameters.xml). If only the file name is specified, the command looks for the file in: VBS4 Installation\systems\compositors.

    Return Value:

    Nothing

    VBS4 Examples (VBS4 22.2+)

    Examples:

    		 
        
setRenderMasking     
[true, "../../classification.xml"]

    VBS3 Description

    Description:

    Controls the visibility of a video mask, and defines its alpha and color settings.

    The mask covers all elements that are defined in setRenderMasked with the specified, solid color. The setting is persistent between mission restarts.

    After applying the command, all the transparent objects, masked or not, become either fully opaque or fully transparent, depending on the alpha parameter.

    VBS3 Syntax

    Syntax:

    setRenderMasking [mask, alpha, color]

    Parameters:

    • mask: Boolean - If true, then masking effects are applied to the objects specified in setRenderMasked. If false, then any existing effects are turned off.

    • alpha: Number - A reduction factor applied to alpha effects (for example, smoke or glass). Range is from 0 to 1, making objects with lower alpha value fully transparent and with higher alpha value fully opaque. For example, a value of 0 makes smoke the most visible and largest, and 1 makes it invisible.

    • color: Color - Color that is applied to the mask.

    Return Value:

    Nothing

    VBS3 Examples

    Examples:

    		 
    Ground is masked in red, and a 0.1 alpha factor applied to the smoke grenade:

    setRenderMasking [true, .1, [255,0,0]];
    "ground" setRenderMasked true;
