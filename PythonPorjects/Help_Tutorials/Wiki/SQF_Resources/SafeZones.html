    SafeZones

    In order to easily accommodate different screen ratios when working with dialogs, the concept of "safeZones" has been introduced.

    At a ratio of 4:3 (e.g. 1024x768 or 1600x1200) the screen locations are [0,0] at the upper left, and [1,1] at the lower right corner.
    At other ratios though (e.g. 16:9), these positions would NOT be at the outermost edges anymore, but slightly offset. By using the different SafeZone variables, these offsets can be used in a dialog, so that its position and size will always be in the same relation to the absolute screen area.

    The following values are available:

    • safeZoneX: Leftmost edge of screen (middle screen in case of TripleHead). Normally a negative value that indicates how far left of [0,0] the absolute edge is.

    • safeZoneXAbs: Same as safeZoneX, but leftmost screen for TripleHead displays.

    • safeZoneY: Uppermost edge of screen. Normally a negative value that indicates how far above [0,0] the absolute edge is.

    • safeZoneW: Total width of screen. Same as 1 + 2*abs(safeZoneX).

    • safeZoneWAbs: Same as safeZoneW, but leftmost screen for TripleHead displays.

    • safeZoneH: Total height of screen. Same as 1 + 2*abs(safeZoneY).

    Be aware that if a dialog uses safeZone commands, and the resolution is changed while the dialog is open, that the values are not changed dynamically. In order to apply the new values (based on the new resolution/aspect ratio), the dialog has to be closed and re-opened.

    Typical positions

    Some examples for specific screen locations, and the typical syntax used to get them, as well as the actual values returned for 4:3 and 16:9 ratios:

    • Upper left corner: [safeZoneX,safeZoneY]
      For 4:3 ratios this would return [0,0], for 16:9 ratios it would be [-.167,0], as there is an extra horizontal margin in widescreen displays.

    • Screen center: [0.5,0.5]
      No matter the resolution, the center of the screen is always at [0.5,0.5], as the area from [0,0] to [1,1] itself is always centered.

    • Lower right corner: [safeZoneW-abs(safeZoneX), safeZoneH-abs(safeZoneY)]
      Take the overall screen width (safeZoneW) and subtract from it the width of the extra margin (abs safeZoneX). Do the equivalent for the heights.


    Area sizing & positioning

    Controls with the specified properties will cover the described areas:

    • Whole screen (middle screen in case of TripleHead):

    x = safezoneX;
    y = safezoneY;
    w = safezoneW;
    h = safezoneH;

    • All screens in TripleHead:

    x = safezoneXAbs;
    y = safezoneY;
    w = safezoneWAbs;
    h = safezoneH;

    • Centered area with 80% width and height:

    x = safezoneX + 0.1 * safezoneW;
    y = safezoneY + 0.1 * safezoneH;
    w = safezoneW * 0.8;
    h = safezoneH * 0.8;

    • Area in upper left corner of the screen (25% of overall screen width/height):

    x = safezoneX;
    y = safezoneY;
    w = safezoneW/2;
    h = safezoneH/2;

    • Area in lower right corner of the screen (25% of overall screen width/height):

    x = safezoneW - safezoneW/4;
    y = safezoneH - safezoneH/4;
    w = safezoneW/4;
    h = safezoneH/4;
