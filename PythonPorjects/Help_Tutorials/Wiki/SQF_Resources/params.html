    params

    Introduced in

    Version:

    VBS4 23.1.0

    Description

    Description:

    Parses the input argument as an array of private variables. When the command is used without an input argument, as shown in main syntax, the internal variable _this is used as the argument.

    In addition to parsing as variables, the input can be tested in case it is undefined, of the wrong type, or of the wrong size (in case of an array), and substituted if necessary with default values.

    All variables names must start with an underscore and be enclosed in double quotes:

    params ["_myVar1", "_myVar2"];

    It is a good practice to make your local variables private (through private or params ) in order to avoid overwriting a local variable of the same name.


    Syntax

    Syntax:

    params [element1, element2, ...elementN]

    Parameters:

    elementN: String or Array:

    • String - Name of the private variable (must start with an underscore, for example: "_myVar").

    • Array - In the format [variableName, defaultValue, expectedDataTypes, expectedArrayCount]:

      • variableName: String - Name of a private variable (must begin with an underscore, for example: "_myVar").

      • defaultValue: Anything - Default value to return, if the input element is undefined, of the wrong type, or of the wrong size (in case of an array).

      • expectedDataTypes: Array of Data Types - (Optional) Checks if the passed value is one of listed Data Types. If not, a default value is used instead. An empty array [] means that every data type is accepted.

      • expectedArrayCount: Number or Array - (Optional) Single size or array of sizes. If the passed input value is an array, checks that it has a certain number of elements. If not, a default value is used instead. An empty array [] means that any size is accepted.

    Return Value:

    Boolean - Returns false, if an error occurred, or the default value is used; otherwise returns true.

    Alternative Syntax

    Syntax:

    argument params [element1, element2, ...elementN]

    Parameters:

    argument: Anything - An array of parameters is expected. If a non-array argument is passed, it is converted to a 1-element array.

    elementN: String or Array:

    • String - Name of the private variable (must start with an underscore, for example: "_myVar").

    • Array - In the format [variableName, defaultValue, expectedDataTypes, expectedArrayCount]:

      • variableName: String - Name of a private variable (must begin with an underscore, for example: "_myVar").

      • defaultValue: Anything - Default value to return, if the input element is undefined, of the wrong type, or of the wrong size (in case of an array).

      • expectedDataTypes: Array of Data Types - (Optional) Checks if the passed value is one of listed Data Types. If not, a default value is used instead. An empty array [] means that every data type is accepted.

      • expectedArrayCount: Number or Array - (Optional) Single size or array of sizes. If the passed input value is an array, checks that it has a certain number of elements. If not, a default value is used instead. An empty array [] means that any size is accepted.

    Return Value:

    Boolean - Returns false, if an error occurred, or the default value is used; otherwise returns true.

    Examples

    Example 1:
    [1, 2, 3] call {
    private ["_one", "_two", "_three"];
    _one = _this select 0;
    _two = _this select 1;
    _three = _this select 2;
    // ...
    };
     
    // Same as the preceding code, only using the params command
    [1, 2, 3] call {
    params ["_one", "_two", "_three"];
    // ...
    };

    Example 2:
    [123] call {
    params ["_myvar"];
    };
     
    // The following produces the same result as the preceding code
    123 call {
    params ["_myvar"];
    };

    Example 3:
    position player params ["", "", "_z"];
    if (_z > 10) then {
    hint "YOU ARE FLYING!";
    };

    Example 4:
    // All three variables are made private, but only _var1 and _var3 are defined
    [1, nil, 2] params ["_var1", "_var2", "_var3"];
     
    // All three variables are private and defined
    [1, nil, 2] params ["_var1", ["_var2", 23], "_var3"];

    Example 5:
    // The following hint shows [1, 2, true]
    // The script exits, a default value is used due to a missing value
    [1, 2] call {
    if (!params ["_var1", "_var2", ["_var3", true, [true]]]) exitWith {
    hint str [_var1, _var2, _var3];
    };
    };
     
    // The following hint shows [1,2,true]
    // Script exits, default value was used due incorrect value type
    [1, 2, 3] call {
    if (!params ["_var1", "_var2", ["_var3", true, [true]]]) exitWith {
    hint str [_var1, _var2, _var3];
    };
    };

    Example 6:
    // Passes validation
    [1, "ok", [1, 2, 3]] call {
    if (!params [
    ["_var1", 0, [0]],
    ["_var2", "", [""]],
    ["_var3", [0,0,0], [[], objNull, 0], [2,3]]
    ]) exitWith {};
    hint "ok";
    };
     
    // Fails, because the passed array is expected to be of 0 length (empty)
    [1, 2, [3, 4, 5]] call {
    if (!params ["_var1", "_var2", ["_var3", [], [[], objNull, 0], 0]]) exitWith {};
    hint "ok";
    };

    Example 7:
    [1, 2, 3, [4, 5, 6]] call {
    params ["_one", "_two", "_three"];
    _this select 3 params ["_four", "_five", "_six"];
    };

    Example 8:
    {
    _x params ["_group", "_index"];
    // ...
    } forEach waypoints group player;
     
    fn_someFnc = {
    // Extract the x, y, and z from the _position array
    params ["_position", ["_direction", 0], ["_name", ""]];
    _position params ["_x", "_y", "_z"];
    // ...
    };
     
    [position player, direction player, name player] call fn_someFnc;

    Example 9:
    player addEventHandler ["HitPart", {
    _this select 0 params ["_target", "_shooter", "_projectile"];
    }];
