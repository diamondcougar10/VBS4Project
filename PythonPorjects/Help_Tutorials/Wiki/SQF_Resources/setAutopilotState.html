    setAutopilotState

    Introduced in

    Version:

    VBS3 22.1.0, VBS4 21.1.0

    Description

    Description:

    Sets the state of VBS Control autopilot unit for the given crew position in a vehicle.

    VBS Control autopilot is an invisible unit with VBS Control brain, which is able to take control of the crew position. It can run a Behavior Tree (setBT) and receive messages (receiveMessage).

    This command can be used to switch between autonomous driving (driven by a VBS Control autopilot unit) or normal driving (driven by a unit in the driver seat - the unit can be Legacy AI (VBS3) or Game AI (VBS4 22.2+) / Control AI / Player).

    Syntax

    Syntax:

    veh setAutopilotState [crewPositionType, state]

    Parameters:

    • veh: Object - Vehicle to set the autopilot state to.

    • crewPositionType: String - Type of crew position. Can be:

      • "driver" - Driver position. The "driver" autopilot unit can drive a vehicle the same way as a VBS Control Actor in the driver seat.

    • state: String - State of the autopilot unit. Can be:

      • "NoAutopilot" - No autopilot is present. Deletes the autopilot unit if it is present.

      • "AutopilotDisabled" - Autopilot is present, vehicle control is disabled. Creates the autopilot unit if it is not present.

      • "AutopilotEnabled" - Autopilot is present, vehicle control is enabled. Creates the autopilot unit if it is not present.

    Return Value:

    Nothing

    Examples

    Examples:

    		 
    // Make any vehicle drive autonomously by creating VBS Control autopilot
    _veh setAutopilotState ["driver", "AutopilotEnabled"]; // Create a VBS Control autopilot unit controlling the driver position
    _autopilot = _veh getAutopilot ["driver"]; // Get the newly created autopilot unit
    _r = _autopilot setBT ["drivingBehavior", "autopilotRoot"]; // Assign a behavior tree to it
    _autopilot receiveMessage ["Move", ["position", getPos player]]; // Send a message to drive the vehicle to a player position

    // Switch control of the vehicle to the actual driver sitting in the driver seat
    _veh setAutopilotState ["driver", "AutopilotDisabled"]; // VBS Control autopilot unit can still evaluate the assigned Behavior Tree, but cannot control the vehicle

    // Make the vehicle normal again
    _veh setAutopilotState ["driver", "NoAutopilot"]; // Delete VBS Control autopilot unit

    Additional Information

    See also:

    getAutopilotState, getAutopilot

    Multiplayer:

    If the vehicle is not local, it creates the autopilot where the command is issued, and then switches the locality of the autopilot to the instance where the vehicle is local.
