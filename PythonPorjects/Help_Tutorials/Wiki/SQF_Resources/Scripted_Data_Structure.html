    Scripted Data Structure

    Scripted data structures are custom data structures that can be defined for use with the following SQF struct functions:

    • fn_vbs_newStructure

    • fn_vbs_setStructValue

    • fn_vbs_getStructValue

    The structures must be defined as configuration classes within CfgScriptDataStructs. A data struct definition has one or more members that each have a defined value type and default value. The SQF struct functions enforce that every member in a given structure is present and has some value, and that every member has a value of the defined type.

    Configuration Definition

    A new struct type is defined as a class within class CfgScriptDataStructs. The class name is used to identify the struct type. Members are defined within sub-class members, where each member is its own class. The member class name defines the name of that member, and within the member class the data type and default value are defined.

    Here is an example of the complete structure:

    class CfgScriptDataStructs
    {
    class MyDataStruct // Struct type name
    {
    class members
    {
    class description // Member name
    {
    type = "string";
    defaultValue = "Hello World!";
    };
    class boolMember
    {
    type = "bool";
    defaultValue = true;
    };
    class numberValue
    {
    type = "scalar";
    defaultValue = 3.1415;
    };
    };
    };
    };

    Macros

    While the previous example shows the configuration structure in its true form, it is recommended to use a macro to define each member on a single line, to make it more readable.

    Here is an example using the same structure from the previous example:

    // Macro to define a structure member
    #define STRUCT_MEMBER(m_name,m_type,m_defval) \
    class m_name \
    { \
    type = m_type; \
    defaultValue = m_defval; \
    }
     
    class CfgScriptDataStructs
    {
    class MyDataStruct
    {
    class members
    {
    STRUCT_MEMBER(description,"string","Hello World!");
    STRUCT_MEMBER(boolMember,"bool",true);
    STRUCT_MEMBER(numberValue,"scalar",3.1415);
    };
    };
    };

    Member Data Types

    The following default data types are supported for members:

    • scalar - Number values.

    • bool - Boolean values.

    • string - String text values.

    • array - Array values, default value must be defined as a string (for example, "[1, 2, 3, 4, 5]").

    • object - Default value is always objNull, and can be defined as an empty string in the configuration.

    In addition to these default data types, a member type can also be defined as another struct type. This allows nesting of structures inside of other structures. For example:

    ...
    class MySuperStruct
    {
    class members
    {
    STRUCT_MEMBER(superDescription,"string","I am a super-structure");
    STRUCT_MEMBER(childStruct,"MyDataStruct","['MyDataStruct'] call fn_vbs_newStructure"); // Invoke newStructure function to populate childStruct
    };
    };

    In this example, MySuperStruct contains a MyDataStruct as defined in the previous examples as a member called childStruct.

    The default value for a struct member type can be left empty, in which case, that member is initialized to a nil value. In order to initialize the member with a nested struct, the default value should be entered as a code expression that creates the structure as seen in the previous example. Also, be aware that the default value expression only gets executed, if the structure is not supplied with an initial value for that member when it is instantiated.

    Struct Inheritance

    Configuration inheritance can be used when defining the classes for a structure. However, it is important to note that by defining a struct class to inherit from another struct class, it also defines the new class as a sub-type of the inherited type, when enforcing member struct types. So, with the preceding examples, if you define a new structure called MySubStruct that inherits from MyDataStruct, then the childStruct member of MySuperStruct can also hold a MySubStruct instance, instead of a MyDataStruct instance. This is based on the assumption that the inherited class still contains all the members of its base type. Therefore, it is important to make sure that all the base-class members are also inherited, and not overridden. The inheriting struct can have additional members that the base struct does not.

    So, for example, the configuration should look like this:

    ...
    class MySubStruct: MyDataStruct
    {
    class members: members // This part is very important - if not done, then the original members from MyDataStruct are not inherited, which breaks the subtype rule
    {
    STRUCT_MEMBER(objectReference,"object","");
    };
    };

    Using Structs in SQF

    Once a data struct is properly defined in a configuration namespace, it can be instantiated in scripts, using the fn_vbs_newStructure function. The new structure will have all members populated with default values, unless other values are specified in the fn_vbs_newStructure function parameters. Member values can be set with fn_vbs_setStructValue, and queried with fn_vbs_getStructValue.

    Here is an example of utilizing the structure types from the preceding configuration examples:

    local _childStruct = ["MySubStruct"] call fn_vbs_newStructure;
    local _parentArgs = [];
    [_parentArgs, "childStruct", _childStruct] call fn_vbs_setKeyValue;
    local _parentStruct = ["MySuperStruct", _parentArgs] call fn_vbs_newStructure;
    [_parentStruct, "superDescription", "Parent structure"] call fn_vbs_setStructValue;
    [_childStruct, "objectReference", player] call fn_vbs_setStructValue;
     
    waitUntil
    {
    _childStruct = [_parentStruct, "childStruct"] call fn_vbs_getStructValue;
    _childObject = [_childStruct, "objectReference"] call fn_vbs_getStructValue;
    if (isNil "_childObject") then
    {
    _childObject = objNull; // Child struct must be of type MyDataStruct and not MySubStruct
    };
    !(alive _childObject);
    };
     
    if ([_childStruct, "boolMember"] call fn_vbs_getStructValue) then
    {
    hint format
    [
    "Struct description: %1\nNumber: %2",
    [_childStruct, "description"] call fn_vbs_getStructValue,
       [_childStruct, "numberValue"] call fn_vbs_getStructValue
    ];
    };

    Querying the Struct Type

    It is also possible to query what type a struct is by using the reserved member key _type in fn_vbs_getStructValue. For example:

    local _structType = [_struct, "_type"] call fn_vbs_getKeyValue;
