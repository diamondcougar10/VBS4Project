    doWatch


    This AI command / function works for Game AI and Control AI in VBS4. For any differences in the behavior between Game AI and Control AI, see the command / function AI Parity section.

    Introduced in

    Version:

    1.00

    Description

    Description:

    Orders the given units to watch the given position or target (without radio messages). Use objNull as the target to order the unit(s) to stop watching the position / target.

    Syntax

    Syntax:

    units doWatch position

    Parameters:

    • units: Object or Array - Units ordered to watch.

    • position: Array - format Position-2 - Position to watch.

    Return Value:

    Nothing

    Alternative Syntax

    Syntax:

    unit doWatch target

    Parameters:

    • units Object or Array - Units ordered to watch.

    • target: Object - Target to watch.

    Return Value:

    Nothing

    Examples

    Examples:

    		 
    The unit named "soldierOne" watches the position, where the marker "MarkerMoveOne" is placed:
    _soldierOne doWatch markerPos "MarkerMoveOne"

    The unit named "soldierOne" watches the unit named "eastSoldier":
    _soldierOne doWatch _eastSoldier

    Additional Information

    See also:

    commandWatch

    Multiplayer:


    Problems:


    AI Parity:

    Game AI considerations:

    • Not functional.

    Control AI considerations:

    • Does not work for AV-enabled vehicles (with the Autopilot option enabled - see AV Mission Preparation in the Introduction to VBS Guide).

    • Watching is done by rotating the whole body (no tracking by head movement only).

    • Vertical watching is not implemented yet (the watchman does not look up or down, if there is an altitude difference).

    • Multiple units can be commanded at once (they can be in different groups).

      [unit_1, unit_2, unit_3] doWatch target;

    • The command makes the unit watch the given target or position indefinitely. No other behavior logic runs, while the command is being performed (for example, no reaction to contact happens).

    • Cannot be combined with other doX commands either. If you run doWatch on a unit running a doMove, the unit stops.

    • If a unit in a vehicle is commanded to watch, only that unit receives the command (in this case, any orders that are already being executed do not get cancelled). If the driver of a vehicle is commanded to watch, the vehicle does not complete its previous order, but stops and watches the target. If the entire vehicle is commanded to watch, the vehicle cancels its previous orders and executes the command.

    • Does not accept objNull as a target to stop watching.
