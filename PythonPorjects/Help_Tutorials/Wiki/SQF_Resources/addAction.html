    addAction


    Introduced in

    Version:

    1.20

    Description

    Description:

    Add an action to an object. The action is usable by the object itself, as well as anyone who is within proximity of the object and is looking at it.

    If an action is added to a unit, the action is only visible when the unit is on foot. If the unit enters a vehicle, the action is not visible either to the unit, or to anyone in / near the unit vehicle. To make the action stay visible even after the unit enters a vehicle, use fn vbs addActionEx.

    fn_vbs_addActionEx is deprecated by the addAction VBS4 Alternative Syntax (VBS4 23.2+). To place an action that should be visible inside vehicles, the addAction command should be used in VBS4.

    The action runs a script, which has the following parameters passed to it in the _this variable:

    [target, caller, ID, data]

    • target: Object - Object which the action is assigned to.

    • caller: Object - Unit that activates the action.

    • ID: Number - Index of the activated action (used to remove the action with removeAction).

    • data: Anything - Extra arguments passed to the addAction command (see Syntax).

    Syntax

    Syntax:

    holder addAction [title, filename, arguments, priority, showWindow, hideOnUse, shortcut, condition]

    Parameters:

    • holder: Object - Object to assign the action to.

    • title: String - Action name which is displayed in the Action Menu.

    • filename: String - Path to the script (relative to the mission folder) that is called when the action is activated.

    • arguments: Anything - Extra data to pass to the script (uses _this select 3 for the script) (optional).

    • priority: Number - Priority value of the action (determines how high it appears in the Action Menu). Actions are arranged in descending order according to this. Every game action has a preset priority value, which can be negative. Actions with the same values are arranged in the order in which they were created (newest at the bottom) (optional).

    • showWindow: Boolean - If set true, Action Menu appears on screen, as soon as the action is available. If false, the action is still added to the Action Menu, but does not cause it to appear, if hidden (optional).

    • hideOnUse: Boolean - If set to true, the Action Menu hides itself after the action is used. If false, the Action Menu stays open on the selected action after use (optional).

    • shortcut: String - VBS Key Actions that activate the action. DIK KeyCodes can also be used. Use empty string ("") for no keyboard shortcut (optional).

    • condition: String - Code that must return true for the action to be shown. Variables _target (the object the action is attached to) and _this (executing unit) can be used in the evaluation (optional, default: true).

    Return Value:

    Number - Index of the added action. Used to remove the action with removeAction.

    Alternative Syntax

    Syntax:

    holder addAction [[aim, fov, los, tolerance], title, filename, ...] (V2.00+)

    Parameters:

    • holder: Object - Object to assign the action to.

    • aim: Boolean - If true, then the player cursor must be aimed at the holder (if the object is obstructed, aim is always false, even when the player points at it, and the los check has been disabled).

    • fov: Boolean - If true, then the holder must be within the player field of view (even if disabled, the action disappears if the player view angle is about 90 degrees away from the object).

    • los: Boolean - If true, then the holder must be within the player line of sight (not obstructed).

    • tolerance: Number - Distance (in meters) the holder can be outside of the player line of sight, and still be considered visible (some objects, even small ones, may need a tolerance > 0 in order for los to return true, even if there are no external obstructions between the player and the object).

    • title: String - Action name which is displayed in the Action Menu.

    • filename: String - Path to the script that is called when the action is activated. Relative to the mission folder.

    • ... - All other arguments as described in the Syntax section.

    Return Value:

    Number - Index of the added action. Used to remove the action with removeAction.

    VBS4 Alternative Syntax (VBS4 23.2+)

    Syntax:

    holder addAction [[aim, fov, los, tolerance, inVehicle, radius], title, filename, arguments, priority, showWindow, hideOnUse, shortcut, condition, position, category, enableCondition]

    Parameters:

    • holder: Object - Object to assign the action to.

    • action visibility parameters: Array - Contains visibility parameters for the action (optional, can be set to nil or an empty array, to use the engine default value).

      • aim: Boolean - If true, the player must aim at the holder to make the action available. If there is an obstacle obstructing the shooting line, the action is not displayed, even if the fov parameter is set to true (optional, default: false, can be set to nil to use the engine default value).

      • fov: Boolean - If true, the holder must be within the player field of view to make the action available. Even if disabled, the action disappears if the player view angle is about 90 degrees away from the object (optional, default: false, can be set to nil to use the engine default value).

      • los: Boolean - If true, the holder must be inside the player line of sight and not obstructed to make the action available (optional, default: false, can be set to nil to use the engine default value).

      • tolerance: Number - Distance (in meters) the holder can be outside of the player line of sight, and still be considered visible to spawn the action. Some objects, even small ones, may need a tolerance > 0 to be considered inside the los, even if there are not obstructions (optional, default: 0, can be set to nil to use the engine default value).

      • inVehicle: Boolean - If true, the action is also available inside vehicles (optional, default: false, can be set to nil to use the engine default value).

      • radius: Number - Defines the range in meters from the object to make the action available (optional, default: 10m, can be set to nil to use the engine default value).

    • title: String - Action name displayed in the action list or radial menu.

    • filename: String - Path to the script (relative to the mission folder) that is called when the action is activated. OR Code - Code executed when the action is triggered.

    • arguments: Anything - Extra data to pass to the script. Use _this select 3 to access this arguments list (optional).

    • priority: Number - This parameter is deprecated in VBS4 and is ignored. The best practice is to use nil for it

    • showWindow: Boolean - This parameter is deprecated in VBS4 and is ignored. The best practice is to use nil for it.

    • hideOnUse: Boolean - If set to true, the radial Quick Menu gets hidden after the action is used. If false, it stays open on the selected action after use. This parameter has no effect on 3D World Actions, since they are not in the radial Quick Menu. The best practice is to use nil for it, if needed (optional, default: true).

    • shortcut: String - VBS Key Actions that activate the action. DIK KeyCodes can also be used. Use empty string ("") for no keyboard shortcut.

    • condition: String - Code that must return true for the action to be shown. Variables _target (the object the action is attached to) and _this (executing unit) can be used in the evaluation (optional, default: true).

    • position: String or Position3D - Defines either the memory point designating the source (String) or the relative position to the object center of mass (Position3D) (optional, default: [0,0,0], can be set to nil, an empty array, or an empty string to use the engine default value).

    • category: String - ID handle of the parent category in the Quick Menu. Use the class name string for categories defined in configuration files (see CfgQuickMenu in Quick Menu Classes in the VBS Developer Suite Reference). The default and fallback value is an empty string (which means that the new action is added at the root level of the Quick Menu).

    • enableCondition: String - Code that must return true for the action to be enabled. Disabled actions appear grayed out in the Quick Menu. Variables _target (the object the action is attached to) and _this (executing unit) can be used in the evaluation (optional, default: true).

    Return Value:

    Number - Index of the added action. Used to remove the action with removeAction .

    Examples

    Examples:

    		 
    // Adds "hello" to the player Action Menu. On execution, passes an array of the caller, target, and index to hello.sqf.
    IDindex = player addAction ["Hello", "hello.sqf"];

    hello.sqf:

    _target = _this select 0;
    _caller = _this select 1;
    hint format ["%1 says hello to %2", name _caller, name _target];

    // Adds "hello" to the Action Menu of the player driving vic1. On execution, passes an array to hello.sqf.
    IDindex = vehicle vic1 addAction["Hello", "hello.sqf", [], 6, true, true, "", "driver _target == player"];

    // (VBS4 23.2+) 3D World Action
    // Action assigned to a vehicle called "vehicle1" to display its side when triggered
    vehicle1 addAction
    [
        [
            false, // It is not necessary to aim exactly at the vehicle to see the action
            true, // The vehicle must be within the player FOV to see the action
            true, // The player must have a clear line of sight of the vehicle to see the action
            nil, // Use the engine default value
            false, // The action is not available if the player is in a vehicle
            5 // The action appears when the player is within 5 meters of the vehicle
        ],
        "Inspect vehicle", // Name of the action
        { // Code to execute when the action is triggered
            _vehicleSide = side (_this select 0);
            cutText [format ["Vehicle side: %1", _vehicleSide], "PLAIN DOWN"];
        },
        [], // No arguments are passed
        nil, // Deprecated
        nil, // Deprecated
        nil, // Ignored for 3D World Actions.
        "Jump", // The action can be triggered by pressing the key assigned to Jump
        "(side player) == west", // The action is displayed only to BLUFOR players
        [0, 0, 2] // The action is positioned 2 meters higher from the center of mass
    ];

    // (VBS4 23.2+) Personal action
    // Action assigned to players to remove all their weapons, even when inside a vehicle
    // Visible in the Quick Menu, but grayed out for OPFOR players
    player addAction
    [
      [ // To use the default value here, you can pass either an empty array or nil.
            nil, // Use engine default value
            nil, // Use engine default value
            nil, // Use engine default value
            nil, // Use engine default value
            true // The action is visible even if the player is in a vehicle
            // The radius is not defined, so the default value is used
        ],
        "Remove all weapons", // Name of the action
        { // Code to execute when the action is triggered
            removeAllWeapons player;
        }, 
        [], // No arguments are passed
        nil, // Deprecated
        nil, // Deprecated
        false, // The Quick Menu remains open after triggering the action
        "", // Use the default value (no shortcut)
        "", // Use the default condition value (true)
        "", // Use the default position value (center of mass). To use the default value here, you can pass either an empty array [0, 0, 0], an empty string, or nil.
        "Weapon", // ClassName of the Quick Menu category
        "side player == west" // Condition to allow triggering is set to BLUFOR - OPFOR players still see the action, but it is grayed out
    ];

    Additional Information

    See also:

    removeAction, fn vbs addActionEx, availableActions

    Multiplayer:

    Multiplayer usage of this command can be complicated, because everything about the command is local. This has a number of implications. For example, a given object might have no actions on client A, 1 action on client B, and 2 actions on client C. Each client sees and uses separate actions - nothing is shared across the network.

    This means:

    • The script that runs, when the action is used, only runs on the client that uses the action.

    • Actions are only visible on the computer where they were added. To make an action visible to all clients, the addAction command must be executed on all of them. When doing this, however, the index returned by the command can vary across all clients.

    • Actions are only removed on the computer where the removeAction command is issued.

    This means, for example, that the self-removing Action Menu example in the Examples section would not work properly in a network environment. If one client uses the generator, it would no longer see the generator action. However, all other clients would still see the action, until they themselves use it.
