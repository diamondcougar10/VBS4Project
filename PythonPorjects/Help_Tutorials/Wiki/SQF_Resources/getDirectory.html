    getDirectory


    Introduced in

    Version:

    1.24

    Description

    Description:

    Returns an absolute path to directory specified by the given ID.

    The possible IDs are:

    • 0: User Directory

    • 1: Mission Directory

    • 2: Installation Directory

    • 3: Directory of the VBS executable (which might be different from the installation folder for Buldozer) (V1.33+)

    In VBS4:

    The structure of the mission folder in VBS4 is different from that of VBS3, due to the concept of Battlespaces (see Battlespaces Folder in the Introduction to VBS4 Guide). Depending on the Mode used (Prepare / Preview / Execute, where the Developer Console is available), each ID can have different return values for the absolute path, which also relies on the Microsoft Windows \Documents\ and %LOCALAPPDATA% folders (different for each Microsoft Windows user):

    \Documents\ is the Microsoft Windows Documents folder (for example, C:\Users\User_Name\Documents\).

    %LOCALAPPDATA% is the Windows local application data folder (for example, C:\Users\User_Name\AppData\Local\).

    Main Menu:

    • 0: User Directory - Returns: \Documents\VBS4\

    • 1: Mission Directory - Undefined.

    • 2: Installation Directory - Returns the absolute path to the VBS4 installation folder (for example, C:\Program Files\BISim\VBS4) (the path is not ended with a backslash).

    • 3: Directory of the VBS executable - The VBS4 executable is in the VBS4 installation folder, so returns the absolute path to the VBS4 installation folder (for example, C:\Program Files\BISim\VBS4) (the path does not end with a backslash).

    Prepare Mode (OME):

    • 0: User Directory - Returns: \Documents\VBS4\

    • 1: Mission Directory - Returns: \Documents\VBS4\Battlespaces\Battlespace_Name\Missions\Mission_Name\

    • 2: Installation Directory - Returns the absolute path to the VBS4 installation folder (for example, C:\Program Files\BISim\VBS4) (the path is not ended with a backslash).

    • 3: Directory of the VBS executable - The VBS4 executable is in the VBS4 installation folder, so returns the absolute path to the VBS4 installation folder (for example, C:\Program Files\BISim\VBS4) (the path does not end with a backslash).

    Preview Mode (RTE):

    • 0: User Directory - Returns: \Documents\VBS4\

    • 1: Mission Directory - Returns: %LOCALAPPDATA%\VBS4\Battlespaces\__tmpPREVIEWsave\

    • 2: Installation Directory - Returns the absolute path to the VBS4 installation folder (for example, C:\Program Files\BISim\VBS4) (the path is not ended with a backslash).

    • 3: Directory of the VBS executable - The VBS4 executable is in the VBS4 installation folder, so returns the absolute path to the VBS4 installation folder (for example, C:\Program Files\BISim\VBS4) (the path does not end with a backslash).

    Execute Mode (Hosted RTE):

    • 0: User Directory - Returns: \Documents\VBS4\

    • 1: Mission Directory - Returns: \Documents\VBS4\Battlespaces\Battlespace_Name\Missions\Mission_Name\

    • 2: Installation Directory - Returns the absolute path to the VBS4 installation folder (for example, C:\Program Files\BISim\VBS4) (the path is not ended with a backslash).

    • 3: Directory of the VBS executable - The VBS4 executable is in the VBS4 installation folder, so returns the absolute path to the VBS4 installation folder (for example, C:\Program Files\BISim\VBS4) (the path does not end with a backslash).

    Execute Mode (Server-Connected RTE):

    • 0: User Directory - Returns: \Documents\VBS4\

    • 1: Mission Directory - Returns: \Missions\__CUR_MP\

    • 2: Installation Directory - Returns the absolute path to the VBS4 installation folder (for example, C:\Program Files\BISim\VBS4) (the path is not ended with a backslash).

    • 3: Directory of the VBS executable - The VBS4 executable is in the VBS4 installation folder, so returns the absolute path to the VBS4 installation folder (for example, C:\Program Files\BISim\VBS4) (the path does not end with a backslash).

    In VBS3:

    In V1.5, the directory returned by dirtype of 1 (mission folder) is a temporary path, and always points to the same location (except for unpacked MP scenarios). For packed (exported) missions in training or networking scenarios, the path is relative, but can nevertheless be used in commands that expect an absolute path (for example, in loadBiotope).

    • Editor: C:\Users\(username)\Documents\VBS3\mpmissions\__tmpPreviewDir\__tmpPREVIEWsave.Intro\
      If the mission has a name (has been saved already), the absolute path can be retrieved using: format["%1MPmissions\%2.%3",getDirectory 0,missionName,worldName]

    • Training Scenario: missions\__cur_sp.intro\

    • Network Scenario (packed): mpmissions\__cur_mp.Intro\

    • Network Scenario (unpacked): C:\Users\(username)\Documents\VBS3\mpmissions\(missionname)\

    Syntax

    Syntax:

    getDirectory dirtype

    Parameters:

    • dirtype: Number - Directory type. Possible values are:

      • 0: User Directory

      • 1: Mission Directory (see Description)

      • 2: Installation Directory

      • 3: Directory of the VBS executable (which might be different from the installation folder for Buldozer) (V1.33+)

    Return Value:

    String - Absolute path. The returned path ends with a backslash for dirtype of 0 and 1, but not for dirtype of 2 and 3.

    Examples

    Examples:

    		 
    _userPath = getDirectory 0
    // Might return: C:\Users\SomeUser\Documents\VBS3\
